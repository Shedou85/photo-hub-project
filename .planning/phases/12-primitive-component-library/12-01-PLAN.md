**Step 1: Install clsx**

```bash
cd frontend && npm install clsx
```

clsx is a 1KB utility for conditional className composition. It replaces string concatenation and ternary patterns currently used across pages.

**Step 2: Create `frontend/src/components/primitives/` directory**

**Step 3: Create Button.jsx**

Create `frontend/src/components/primitives/Button.jsx` with:

- JSDoc comment block with `@param` for all props and two `@example` tags (primary variant, secondary variant)
- Function declaration `function Button(...)` (not arrow function, per codebase convention)
- Props: `variant` (default 'primary'), `size` (default 'md'), `children`, `disabled` (default false), `fullWidth` (default false), `className`, `type` (default 'button'), and `...props` spread
- `type` defaults to `'button'` NOT `'submit'` to prevent accidental form submission (Pitfall 1 from research)
- Base classes: `'inline-flex items-center justify-center gap-2 font-semibold transition-opacity duration-150 cursor-pointer disabled:opacity-50 disabled:cursor-not-allowed font-sans'`
- Variant classes object:
  - `primary`: `'bg-[linear-gradient(135deg,#3b82f6,#6366f1)] text-white hover:opacity-90 border-none'`
  - `secondary`: `'bg-blue-50 text-blue-600 border border-blue-200 hover:bg-blue-100'`
  - `danger`: `'bg-red-50 text-red-600 border border-red-200 hover:bg-red-100'`
  - `ghost`: `'bg-transparent text-gray-700 hover:bg-gray-50 border-none'`
- Size classes object:
  - `sm`: `'text-xs py-1.5 px-3'`
  - `md`: `'text-sm py-2.5 px-5'`
  - `lg`: `'text-base py-3.5 px-7'`
- Border-radius: use `rounded-sm` (matches existing button pattern from Phase 11)
- Use `clsx(baseClasses, variantClasses[variant], sizeClasses[size], fullWidth && 'w-full', className)` for className composition
- Render `<button>` element with type, className, disabled, onClick spread from ...props
- `export default Button;`

Do NOT use PropTypes (not installed, eslint rule is off). Use JSDoc only for documentation.

**Step 4: Create Card.jsx**

Create `frontend/src/components/primitives/Card.jsx` with:

- JSDoc comment block with `@param` for all props and two `@example` tags (default card, card with noPadding)
- Function declaration `function Card(...)`
- Props: `children`, `className` (optional), `noPadding` (default false)
- Base classes: `'bg-white border border-gray-200 rounded'` (uses Phase 11 design token `rounded` = 10px)
- Padding class: `noPadding ? '' : 'px-6 py-5'` (matches existing card pattern across all pages)
- Use `clsx(baseClasses, paddingClass, className)` for className composition
- Render `<div>` with className and children
- `export default Card;`

**Step 5: Create Badge.jsx**

Create `frontend/src/components/primitives/Badge.jsx` with:

- JSDoc comment block with `@param` for all props and two `@example` tags (SELECTING status, DELIVERED status)
- Function declaration `function Badge(...)`
- Props: `status`, `children`, `className` (optional)
- Base classes: `'inline-block text-[10px] font-bold uppercase tracking-wider px-2 py-0.5 rounded-full'` (matches existing badge pattern exactly -- text-[10px] is deliberate per Phase 11 decision)
- Status color classes object:
  - `DRAFT`: `'bg-gray-100 text-gray-600'`
  - `SELECTING`: `'bg-blue-100 text-blue-700'`
  - `REVIEWING`: `'bg-green-100 text-green-700'`
  - `DELIVERED`: `'bg-purple-100 text-purple-700'`
  - `DOWNLOADED`: `'bg-purple-200 text-purple-800'`
- Fallback: if status is not in the map, render with `'bg-gray-100 text-gray-600'` (Pitfall 4 from research)
- Use `clsx(baseClasses, statusClasses[status] || statusClasses.DRAFT, className)` for className composition
- Render `<span>` with className and children
- `export default Badge;`

**Important conventions to follow:**
- Use `import clsx from 'clsx';` at top (external library import order)
- Use function declarations not arrow functions for components
- Use `export default ComponentName;` at end of file (codebase convention)
- No PropTypes imports -- project does not use them
- All Tailwind utility classes, no inline styles
- Keep gradient as arbitrary value `bg-[linear-gradient(135deg,#3b82f6,#6366f1)]` per Phase 11 decision
